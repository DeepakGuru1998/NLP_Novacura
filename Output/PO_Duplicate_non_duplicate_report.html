Duplicate Orders and Non-Duplicate Orders Report Summary¶

In this report we go through some general trend of Duplicate Orders and show some valuable insights. Also we will conver some high level stats on the correct orders.¶

Definition (Exact Match Duplicate):
Exact match duplicates are orders that are identical across all key fields. This includes the vendor name, product list, item codes, quantities, and other relevant metadata.

   Scenario:
   These are usually the result of customers submitting the same email or form multiple times, either due to a mistake or a system glitch.

   Detection Logic:
   Compare rows across all relevant fields (e.g., customer_name, po_number, item_description, item_code, quantity).
   Identify rows that are completely identical or repeated.

Definition (Fuzzy Match Duplicates):
Fuzzy match duplicates are orders that are not identical but are very similar, with minor differences like spelling variations, formatting inconsistencies, or shuffled product listings.

   Scenario:
   These often arise when:    

  • A customer submits the same order with slight modifications (e.g., typo correction).    
  • Orders are manually typed or translated, leading to inconsistent representations.    
  • The item list is the same but the order is changed (e.g., reordered product lines).

    •    Detection Logic:
       Use fuzzy string matching (e.g., Partial Token Set Ratio from fuzzywuzzy) on combinations of fields like customer_name, product_name, and item_code which calculates the similarity of two strings by calculating the Levenshtein distance.

       Define a similarity threshold (e.g., ≥ 90%) to flag potential duplicates.
